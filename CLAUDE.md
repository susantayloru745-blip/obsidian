# Wiki Schema — Claude Code 操作手册

本文件是 Claude Code 的工作指南。每次新会话开始时先读此文件，理解知识库结构和工作流程后再操作。

---

## 知识库结构

```
vault/
├── CLAUDE.md          ← 本文件，LLM 操作手册
├── raw/               ← 原始资料（只读，LLM 不修改）
│   ├── assets/        ← 图片、附件
│   └── *.md           ← 剪藏的文章、论文、笔记原文
├── wiki/              ← LLM 维护的结构化知识库
│   ├── entities/      ← 人物、机构、产品页面
│   ├── concepts/      ← 概念、方法论、框架页面
│   ├── sources/       ← 每篇原始资料的摘要页
│   ├── analysis/      ← 分析、比较、综合推理页面
│   └── daily/         ← 每日笔记、日志
├── index.md           ← 全库目录（每次 ingest 后更新）
└── log.md             ← 操作日志（只追加，不修改历史）
```

---

## 页面格式规范

### 所有 wiki 页面必须有 YAML frontmatter

```yaml
---
title: 页面标题
type: entity | concept | source | analysis | daily
tags: [标签1, 标签2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [相关原始资料文件名]
related: [相关wiki页面链接]
---
```

### 页面类型说明

| 类型 | 目录 | 说明 |
|------|------|------|
| `source` | wiki/sources/ | 每篇原始资料的摘要，1篇资料对应1个页面 |
| `entity` | wiki/entities/ | 人物、机构、产品、项目 |
| `concept` | wiki/concepts/ | 概念、理论、方法论、框架 |
| `analysis` | wiki/analysis/ | 比较分析、综合推理、你提问的答案 |
| `daily` | wiki/daily/ | 日常记录，格式 YYYY-MM-DD.md |

## 操作流程

### 1. Ingest（消化新资料）

用户把资料放入 `raw/` 后，执行以下步骤：

1. **阅读**原始资料，提取关键信息
2. **与用户讨论**重点和侧重方向
3. **创建** `wiki/sources/` 摘要页，包含：
   - 核心论点（3-5条）
   - 关键数据/事实
   - 与现有知识的联系
   - 矛盾或值得深思的点
4. **更新**相关 entity 和 concept 页面（可能涉及 5-15 个页面）
5. **更新** `index.md`
6. **追加** `log.md` 记录

### 2. Query（回答问题）

1. 先读 `index.md` 找到相关页面
2. 读取相关 wiki 页面
3. 综合回答，附带页面引用
4. **重要**：有价值的分析结果保存为 `wiki/analysis/` 页面，不让它消失在对话记录里

### 3. Lint（健康检查）

定期执行，检查：
- 孤立页面（无入链）
- 过时声明（被新资料推翻）
- 有提及但缺页面的重要概念
- 缺失的交叉引用
- 数据空白（可用网络搜索填补）

## 写作规范

- 语言：中文为主，专有名词保留英文
- 交叉引用格式：`[[页面名称]]`
- 外部链接：`[显示文字](URL)`
- 引用原始资料：`> 引文内容 — [[sources/资料名]]`
- 不确定的内容标注：`> ⚠️ 待验证`
- 与现有内容矛盾时标注：`> ⚡ 与 [[页面名]] 存在矛盾`

---

## 重要原则

1. **raw/ 目录只读**，LLM 只读取，不修改、不删除
2. **log.md 只追加**，不修改历史记录
3. **每次 ingest 必须更新 index.md**
4. **有价值的分析要落地成页面**，不能只存在于对话
5. **发现矛盾要标注**，不能选择性忽视
6. **用户主导方向**，LLM 负责执行和维护

## 快捷指令

用户可以用以下指令触发标准流程：

| 指令 | 动作 |
|------|------|
| `/ingest [文件名]` | 消化指定原始资料 |
| `/query [问题]` | 查询并回答，结果存入 analysis |
| `/lint` | 执行健康检查 |
| `/daily` | 创建今日日记页面 |
| `/status` | 显示知识库当前统计 |
